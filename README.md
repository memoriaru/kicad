# KiCad S-expression to JSON5 Compiler

A Rust library and CLI tool for converting KiCad schematic files (`.kicad_sch`) from S-expression format to JSON5 format.

## Why JSON5?

- **Human-readable**: Supports comments, unquoted keys, trailing commas
- **Machine-friendly**: Structured data, easy to parse
- **Token-efficient**: ~40-50% fewer tokens than S-expression
- **AI-friendly**: Clean structure makes it easier for AI tools to understand

## Installation

```bash
# Build from source
cd AtopileTest
cargo build --release

# Or install binary
cargo install --path ~/.local/bin
```

## CLI Usage

### Basic Conversion

```bash
# Convert file to stdout
kicad-json5 input.kicad_sch

# Convert file to output file
kicad-json5 input.kicad_sch -o output.json5
```

### Options

| Option | Description |
|--------|-------------|
| `-o, --output` | Output file path |
| `-i, --indent` | Indentation size (default: 2) |
| `--no-comments` | Exclude comments from output |
| `--validate` | Validate input only (no output) |
| `--debug-ast` | Print parsed AST for debugging |
| `-v, --verbose` | Verbose output |

### Examples

```bash
# Verbose conversion
kicad-json5 schematic.kicad_sch -v -o schematic.json5

# Validate only
kicad-json5 schematic.kicad_sch --validate

# Debug AST
kicad-json5 schematic.kicad_sch --debug-ast
```

## Library Usage

```rust
use kicad_json5::{convert_file, convert_str, Json5Generator, Lexer, Parser};

// Convert file
convert_file("input.kicad_sch", "output.json5")?;

// Convert string
let json5 = convert_str("(kicad_sch (version \"20211123\"))")?;

// Use components directly
let lexer = Lexer::new(source);
let mut parser = Parser::new(lexer);
let schematic = parser.parse()?;
let generator = Json5Generator::new();
let json5 = generator.generate(&schematic)?;
```

## Output Format

The input:
```lisp
(kicad_sch (version "20211123") (generator eeschema)
  (uuid "abc-123")
  (title_block
    (title "My Schematic")
    (date "2024-01-01"))
  (lib_symbols
    (symbol "Device:R"
      (pin_numbers hide)
      (in_bom yes)))
  (symbol (lib_id "Device:R") (reference "R1") (value "10k")
    (at 100 50 0)
    (pin (number "1") (net 1))
    (pin (number "2") (net 2))))
```

The output:
```json5
{
  // KiCad Schematic in JSON5 format
  // Generated by kicad-json5 compiler

  version: "20211123",
  generator: "eeschema",

  metadata: {
    // Document identification
    uuid: "abc-123",
    title: "My Schematic",
    date: "2024-01-01",
    paper: "A4",
  },

  // Library symbols definitions
  lib_symbols: [
    {
      id: "Device:R",
      pin_numbers: "hide",
      in_bom: true,
      on_board: true,
    },
  ],

  // Component instances
  components: [
    {
      ref: "R1",
      lib_id: "Device:R",
      value: "10k",
      position: { x: 100, y: 50, rotation: 0 },
      pins: {
        "1": { net: 1 },
        "2": { net: 2 },
      },
    },
  ],
}
```
